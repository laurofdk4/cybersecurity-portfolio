# Botium Toys Internal Security Audit

## Project Overview
This project documents an internal security audit conducted for a fictional company, Botium Toys, as part of the Google Cybersecurity Professional Certificate program.

The objective of this audit was to assess the organizationâ€™s current security posture, identify gaps in controls and compliance, and evaluate potential risks to critical assets using the NIST Cybersecurity Framework (NIST CSF).

## Scope and Objectives
- Review existing technical, administrative, and physical security controls
- Evaluate compliance with relevant regulations and standards, including PCI DSS, GDPR, and SOC
- Identify missing or insufficient controls that could expose the organization to risk

## Frameworks and Standards
- NIST Cybersecurity Framework (CSF)
- PCI DSS
- GDPR
- SOC Type 1 and Type 2

## Key Findings
- Lack of least privilege and access control enforcement
- No disaster recovery or backup strategy in place
- Absence of encryption for sensitive customer data
- Missing intrusion detection system (IDS)
- Weak password policies and no centralized password management

## Outcome
This audit highlights critical areas where Botium Toys must improve its security controls and compliance practices to reduce operational and regulatory risk.

## Disclaimer
This audit is based on a fictional scenario provided for educational purposes.
